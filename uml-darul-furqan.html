Activity Diagram — PPDB Flow
// Step-by-step process from user access to admin decision & notification
●
START — User mengakses website Yayasan Darul Furqan Pariaman
1
User mengklik tombol "Daftar PPDB" di Landing Page
?
IF Sudah punya akun → Login dengan email & password
ELSE → Register akun baru (nama, email, password, nomor HP)
3
SYSTEM IF Register → Sistem kirim email verifikasi ke akun baru
             User klik link konfirmasi di email
4
User masuk ke Applicant Dashboard — klik "Mulai Pendaftaran"
5
STEP 1 Data Siswa
Isi: nama lengkap, TTL, jenis kelamin, alamat, agama, asal sekolah, NISN
6
STEP 2 Data Orang Tua
Isi: nama ayah/ibu/wali, pekerjaan, penghasilan, nomor HP, alamat
7
STEP 3 Upload Dokumen
KK · Akta Kelahiran · Ijazah/SKL · Pas Foto · Rapor (jika diperlukan)
?
IF Format/ukuran file invalid → Sistem tampilkan error → User re-upload
ELSE File valid → Lanjut ke Step berikutnya
8
STEP 4 Review & Submit
Sistem tampilkan ringkasan semua data — User review
?
IF User ingin edit → Kembali ke step terkait
ELSE User klik "Kirim Pendaftaran"
10
SYSTEM Form dikunci (read-only) · Timestamp submission dicatat · Status → Pending
Konfirmasi ditampilkan + email konfirmasi dikirim
11
ADMIN Admin login ke dashboard → melihat daftar pendaftar baru/pending
12
ADMIN Admin buka detail pendaftar → review dokumen satu per satu
?
IF Dokumen tidak valid/lengkap → Admin flag dengan catatan → Notifikasi ke pendaftar untuk re-upload
ELSE Semua dokumen valid → Admin tandai "Verified"
13
ADMIN Admin set status akhir: ✓ Diterima atau ✗ Tidak Diterima
14
Sistem kirim notifikasi final ke pendaftar (email + dashboard)
Pendaftar login → lihat status Diterima ✅ / Tidak Diterima ❌
⬤
END — Proses PPDB selesai
