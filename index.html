II. Activity Diagram
Bagian 2 — Alur Proses PPDB dari Awal hingga Keputusan Akhir
Activity Diagram — Alur Proses PPDB (Penerimaan Peserta Didik Baru)
●
Mulai — Pengguna mengakses website Yayasan Darul Furqan Pariaman
1
Pengguna mengklik tombol "Daftar PPDB" pada halaman utama (Landing Page)
◇
IF Sudah memiliki akun → Login dengan email dan kata sandi
ELSE → Daftar akun baru (nama, email, kata sandi, nomor HP)
3
SISTEM IF Registrasi baru → Sistem mengirim email verifikasi
          Pengguna mengklik tautan konfirmasi di email
4
Pengguna masuk ke Dashboard Pendaftar — mengklik "Mulai Pendaftaran"
5
LANGKAH 1 Data Siswa
Mengisi: nama lengkap, tanggal lahir, jenis kelamin, alamat, agama, asal sekolah, NISN
6
LANGKAH 2 Data Orang Tua / Wali
Mengisi: nama ayah/ibu/wali, pekerjaan, penghasilan, nomor HP, alamat
7
LANGKAH 3 Unggah Dokumen
Kartu Keluarga (KK) · Akta Kelahiran · Ijazah / SKL · Pas Foto · Rapor (jika diperlukan)
◇
IF Format atau ukuran file tidak valid → Sistem menampilkan pesan kesalahan → Pengguna mengunggah ulang
ELSE File valid → Lanjut ke langkah berikutnya
8
LANGKAH 4 Tinjauan & Pengiriman
Sistem menampilkan ringkasan seluruh data — Pengguna melakukan pengecekan akhir
◇
IF Pengguna ingin memperbaiki data → Kembali ke langkah terkait
ELSE Pengguna mengklik "Kirim Pendaftaran"
10
SISTEM Formulir dikunci (baca saja) · Waktu pengiriman dicatat · Status diubah menjadi Menunggu (Pending)
Konfirmasi ditampilkan di layar dan email konfirmasi dikirim kepada pendaftar
11
ADMIN Admin masuk ke dashboard → Melihat daftar pendaftar baru / menunggu verifikasi
12
ADMIN Admin membuka detail pendaftar → Memeriksa dokumen satu per satu
◇
IF Dokumen tidak lengkap / tidak valid → Admin memberikan catatan → Notifikasi dikirim kepada pendaftar untuk mengunggah ulang
ELSE Semua dokumen valid → Admin menandai status "Terverifikasi"
13
ADMIN Admin menetapkan keputusan akhir: Diterima atau Tidak Diterima
14
Sistem mengirim notifikasi hasil akhir kepada pendaftar melalui email dan dashboard.
Pendaftar login dan melihat status: Diterima atau Tidak Diterima
⬤
Selesai — Proses PPDB telah selesai dilaksanakan

Relasi Antar Kelas
User 1 → 0..1 PPDBApplication — (Asosiasi) Satu user memiliki paling banyak satu application PPDB
PPDBApplication 1 → 1 StudentData — (Komposisi) Data siswa merupakan bagian tak terpisah dari application
PPDBApplication 1 → 1 ParentData — (Komposisi) Data orang tua merupakan bagian tak terpisah dari application
PPDBApplication 1 → * Document — (Agregasi) Satu application memiliki banyak dokumen yang diunggah
PPDBApplication memicu → Notification — (Dependensi) Perubahan status application memicu pengiriman notifikasi
User menerima Notification — (Asosiasi) Notifikasi dikirimkan kepada user yang bersangkutan
PPDBPeriod 1 → * PPDBApplication — (Asosiasi) Satu periode PPDB dapat menampung banyak pendaftaran
User (superadmin) mengelola → SchoolProfile · News · PPDBPeriod — (Dependensi) Super Admin bertanggung jawab atas konten dan konfigurasi sistem
